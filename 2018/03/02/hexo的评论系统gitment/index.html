hexo的评论系统gitment

发表于 2018-03-02   |  

我之前在hexo用的是多说系统，今天才知道去年就停止服务了。。所以今天查了一下，换了个基于Github issue系统的Gitment。

  • Gitment：使用 GitHub Issues 搭建评论系统

    作者的说明文档，但是，小白啊，看不懂啊/(ㄒoㄒ)/~~

  • hexo next主题集成gitment评论系统

    刚好我用的也是next主题，直接按部就班的来做就行了。
    不过可能由于版本的不同，里面说的一些路径跟我这里的还是有些差别的，比如说layout/_third-party/comments/这个路径，我这里是在layout/_scripts/_third-party/comments/，等等，不要太死板，要灵活一点，看几个地方。

  • Gitment评论功能接入踩坑教程

    按照小白步骤走完以后，还是出问题了，在我初始化issue的时候提示(这篇文章的第5条)：Error：validation failed
    按照他说的写法还是不对。。。最后我改成了这样：id: '\{\{ page.title \}\}', (去掉\，因为避免hexo编译的时候被处理掉 han!!!)
    这时候才创建成功。

  • 重复的Github issue

    我在不断的初始化每篇文章的评论区的时候，是点进去然后返回，再进下一个，结果我重复进同一篇文章的时候居然都提示没有初始化评论，WTF？
    在issue果真是看到了重复的内容，搞了我好一会儿，本地调、推到Github调，最后才发现居然是本地缓存搞得鬼。。。初始化评论区以后，等会儿再进刷新一下即可。
