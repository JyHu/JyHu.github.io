iOS在桌面及锁屏状态下的推送交互操作

发表于 2016-04-12   |  

iOS的通知，不管是本地通知还是APNS推送通知，在iOS8之前都只有展示的作用，并会提供给开发者一些交互性的操作，但是在iOS8的时候，推出了按钮选择性的交互，在iOS9以后又推出了输入内容的交互。比如微信和短信的下拉回复。
注意：交互操作都在APPDelegate中进行。

UIUserNotificationAction

提供给用户进行交互性操作的类别，这个类只提供了初始化的方法以及一些只读属性，所以它只提供读操作。而实际提供操作的是其子类UIMutableUserNotificationAction

  • identifier : action的唯一标识，用于在系统处理操作事件的时候进行判断
  • title : 显示在通知下拉状态时的按钮上的名称，如果只有输入内容的时候，title无效
  • behavior : 是UIUserNotificationActivationMode枚举类型，用来设置与用户交互的类型，在iOS9以后加入

    UIUserNotificationActionBehaviorDefault 默认的交互类型，只提供一个按钮，进行选择性的交互
    UIUserNotificationActionBehaviorTextInput 提供一个输入框，用以输入内容

  • activationMode : 设置APP响应时的状态，这是一个UIUserNotificationActivationMode枚举类型。

    UIUserNotificationActivationModeForeground : 在响应用户操作的同时会启动APP
    UIUserNotificationActivationModeBackground : 表示系统在响应action的时候，之后在后台执行，并不会进入启动APP

  • parameters : 设置在输入框状态下的时候，回复去人输入完成按钮上的内容
  • authenticationRequired : 表示在响应action操作的时候需不需要验证，主要是在锁频状态下
  • destructive : 在当前action即将销毁的时候是否需要提示

UIUserNotificationCategory

将多个action放在一起，即说明每个通知都可能有多个交互性的操作，但是最多只有两个。
提供操作的是UIMutableUserNotificationCategory

  • identifier : 交互动作类别的唯一标识
  • - (void)setActions:(nullable NSArray<UIUserNotificationAction *> *)actions forContext:(UIUserNotificationActionContext)context;

    actions : 交互动作的数组，其成员为UIUserNotificationAction类型
    context : UIUserNotificationActionContext枚举类型

    UIUserNotificationActionContextDefault : 默认的通知上下文
    UIUserNotificationActionContextMinimal : 一般用这个

UIUserNotificationSettings

通知动作的注册项，只有一个可以操作的方法：

+ (instancetype)settingsForTypes:(UIUserNotificationType)types categories:(nullable NSSet<UIUserNotificationCategory *> *)categories;

types : 枚举类型，表示通知的需要的权限

categories : 通知设置项的类别数组，成员为UIUserNotificationCategory
