iOS BLE Programming 之 PeripheralManager

发表于 2016-05-02   |  

在之前我们说了在iOS的蓝牙开发中作为客户端的CBCentralManager，当时就说过还有作为服务端的CBPeripheralManager等着以后再说，刚好今天有时间就总结一下。

传送门

在上一节中说到了一个CBPeripheral，是蓝牙外设在iOS系统中的一个身份对象，在今天看来呢，就是CBPeripheralManager在中心设备(即客户端-CBCentralManager)中的体现。
那反过来想，当然就能想到，客户端在服务端上也有一种表现形式，那就是CBCentral，每个连接到CBPeripheralManager上的外设都是一个CBCentral，通过对CBCentral的操作，就能实现对单个设备进行数据的通信。
下面是几个比喻：
数据传输载体 -CBCharacteristic :用来互相的进行数据交互，传递设备状态等。CBService是看一看做是一组功能类似的数据载体的集合，有单独的特征和表现形式。
服务端-CBPeripheralManager :所有数据传输载体都是由服务端发出，并在连接成功后和外设进行数据交互。服务端可以向外无目标的广播数据，也能向特定的外设广播数据，同时也能接受客户端写入过来的数据包。
客户端-CBCentralManager :只能在扫描到并且连接成功服务端以后才能使用数据传输的载体，才可以进行数据交互，它可以直接读取或者设置通知监听数据的变动来进行数据的交互操作。它也同时具备接受数据和相连接到的设备写入数据的功能。注意是在成功连接上以后。

服务端中几个重要的概念

  • CBCentral

上面也说了一嘴，它继承自CBPeer，加上自身的一个属性总共也就两个属性，但是它的作用却不小。
一般的蓝牙数据发包是无目标的，就是说所有的外设都能收的到，但是有时候作为点对点的数据传输，传输对象就很重要了，这就是CBCentral。
每一个连接到服务端上得CBCentral都有是唯一的，可以通过它的identifier来进行判断，然后通过设置更新数据的对象来唯一的通知到目标设备，同时，也能在接收到数据的时候进行唯一值得过滤。

  • CBMitableService

继承自CBService，其实看到中间的mutable也就能明白个大概什么意思了。它就是CBService的可变形式，通过它来设置service的属性，然后添加到CBPeripheralManager中进行广播。
具体的参数什么的就那么几个，也不多说了，看看也就能明白了。

  • CBMutableCharacteristic

继承自CBCharacteristic，跟CBMitableService也差不多嘛。看了上一篇文章的就大概能明白什么意思了。

  • CBMutableDescriptor

继承自CBDescriptor。

7、服务端也能接收数据
